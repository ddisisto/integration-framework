How information theory reveals when language models are thinking vs. performing

You’ve felt it. The model gives you three paragraphs of fluent, confident, well-structured prose — and something is off. You can’t point to an error. The grammar is perfect. The structure is sound. But when you finish reading, you don’t know anything you didn’t know before.

You’re not imagining it. And it’s not a vague quality problem. It’s a measurable phenomenon with a precise information-theoretic description, and once you see the mechanism, you can’t unsee it.

The bottleneck nobody talks about

Here’s a fact about transformers that hides in plain sight. At every generation step, the model’s internal state — the full residual stream conditioning the next prediction — lives in a space of thousands of dimensions. The output is a single token from a vocabulary of maybe 100,000 entries.

That’s a massive bandwidth mismatch. The model’s internal representation is vastly richer than what it can say in one token.

Shannon characterised this exact problem in 1948: transmitting a high-entropy source through a bandwidth-limited channel. The solution is sequential coding — spread the message across multiple channel uses. Each token is a channel use. The sequence collectively encodes what no single token could carry.

This isn’t a metaphor. It’s the literal structure of the problem.

And here’s the evidence it’s real: researchers found that simply duplicating the input prompt — repeating the query verbatim so the model processes it twice — improves accuracy by up to 76%. No fine-tuning, no clever engineering. Just giving the model a second pass through the same input. If the bottleneck weren’t a real, active constraint, this trivial intervention would do nothing.

Two regimes, one surface

So the model is squeezing rich internal representations through a narrow channel, one token at a time. Here’s where it gets interesting.

Sometimes each token does genuine work. It surfaces structure that wasn’t available from the preceding context. It bridges domains. It establishes something that changes what’s probable next. Each token opens new territory. I call this the enriching regime.

Sometimes each token reinforces what’s already there. It reproduces known patterns. It settles into familiar trajectories. It restates, in different words, what the context already implies. The output sounds just as good. But it’s not adding structure — just length. This is the degrading regime.

The distinction between these two modes is invisible in everything we normally use to evaluate text. Fluency? Perfect in both. Grammar? Flawless. Coherence? Maintained. The degrading regime doesn’t produce errors. It produces emptiness that looks like substance.

You can actually measure this

Here’s the part that surprised me.

The model’s own surprisal — how unexpected each token is given the preceding context — tracks this distinction directly. When surprisal stays elevated, the model is generating tokens that aren’t predictable from what came before. That’s the enriching regime: each token carries genuine compressive novelty. When surprisal collapses toward zero, the model is producing tokens that are already implied by the context. That’s the degrading regime.

Surprisal is compressive novelty, measured at the token level. It’s not a proxy. It’s the thing itself.

But — and this is the catch — it’s compressive novelty against the model’s own measure of what matters. The model might be enriching by its own standards while producing something useless to you. Whether you’re being enriched depends on whether your sense of what matters aligns with the model’s.

This is the alignment problem, restated at the token level. Two surprisal signals — the model’s and the receiver’s — and the gap between them is where failure lives.

Chain-of-thought solves the same bottleneck

This framework explains something that’s been bugging people about chain-of-thought prompting.

CoT works. Making the model “show its reasoning” before giving a final answer substantially improves performance. The standard explanation is that it “gives the model more computation.” True, but incomplete.

Each intermediate reasoning token enriches the context for the next forward pass. CoT keeps the model in the enriching regime by forcing it to surface structure step by step rather than trying to compress everything through a single pass.

The evidence: that prompt-duplication trick that gives a 76% improvement on standard tasks? It shows minimal additional benefit when chain-of-thought is already active. They’re solving the same problem. CoT already provides the integrative passes that duplication would supply.

But not all chains of thought are equal. A chain where each step surfaces genuinely new structure — elevated surprisal throughout — outperforms a chain that circles, restates, and fills space. The prediction: CoT effectiveness tracks compressive novelty per step, not chain length. More steps only help if each step is actually doing work.

The model must compress — and that shapes everything

One more piece. The model has fewer parameters than its training data has bits. It literally cannot memorise everything. It must find regularities and compress them.

The training objective — next-token prediction — tells it how to compress: preserve what helps predict what comes next. But the training data tells it what to compress. A model trained mostly on code learns that syntactic precision is worth preserving at high fidelity. A model trained mostly on dialogue learns that pragmatic intent matters more.

The dataset doesn’t just determine what the model knows. It determines what the model considers acceptable loss. That’s a value judgement, baked in before deployment, communicated through no explicit instruction.

Why this matters

If you use language models seriously, you’ve developed intuitions about when they’re “on” versus “off” — when the output is genuinely useful versus when it’s elaborate filler. Those intuitions are real, and they’re tracking something measurable.

The enriching/degrading distinction gives that intuition a name. The surprisal measure gives it a number. And the framework behind them — laid out in full here — generates nine testable predictions, several with existing empirical support.

The model can sound brilliant while saying nothing. Now we have vocabulary for when it’s doing which — and the beginning of a way to measure it.
