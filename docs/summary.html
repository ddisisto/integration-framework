What if LLMs are compression engines — and that explains everything?

A summary of Integration

Author
Affiliation

Daniel DiSisto

Independent researcher

Published

March 20, 2026

A transformer’s internal state occupies thousands of dimensions. Its output is a single token from a finite vocabulary. That mismatch isn’t a metaphor — it’s a channel capacity problem Shannon characterised in 1948.

Integration takes this bottleneck seriously and follows it to its consequences.

1 The compression hierarchy

Each layer of a transformer solves a rate-distortion problem: preserve what matters, discard what doesn’t. The key insight is that “what matters” isn’t given by the data — it’s a choice, encoded in the distortion measure. A gzip model of English has learned nothing about English. It has learned which byte sequences recur. A transformer’s deep layers are doing something qualitatively different: building representations whose statistical complexity increases even as mutual information with the raw input is bounded. They’re not just filtering. They’re reorganising.

2 The autoregressive loop

Generation is sequential coding through a bandwidth-limited channel. Each token is a channel use. And this composition is fundamentally ambivalent. When a token surfaces structure unavailable from the preceding context, the loop enriches. When it reinforces existing trajectories — fluent, confident, and adding nothing — it degrades. The distinction is invisible in surface metrics.

This is the paper’s central claim: the difference between integration and mere accumulation. Between depth and verbosity.

The model’s own surprisal provides a direct measure of which regime it’s in — when surprisal collapses, the loop is accumulating, not integrating. But this measures enrichment against the model’s distortion measure. Whether the receiver is being enriched depends on whether their measure aligns — which is the projection bottleneck, encountered again from the measurement side.

3 The surprising evidence

Duplicating a prompt verbatim — just repeating the query so the model processes it twice — improves accuracy by up to 76%. Chain-of-thought reasoning shows no additional benefit from the same trick. The framework explains why: CoT already provides the integrative passes that repetition would supply. They solve the same bottleneck.

4 The alignment reframe

The model’s implicit distortion measure — what it treats as acceptable loss — is set by training. The user’s distortion measure is set by their needs. The gap between these is the alignment problem, restated as a projection bottleneck. The difficulty isn’t capability. It’s that the feedback channels available to close the gap are narrow.

5 Eight testable predictions

The framework generates specific, falsifiable claims — from the scaling profile of statistical complexity across layers, to the prediction that chain-of-thought effectiveness tracks compressive novelty per step rather than chain length. Several already have empirical support; others are untested.
